3. ПОРЯДОК БРОНИРОВАНИЯ
________________________________________________________________________________

3.1. Для оформления бронирования Пользователь обязан:

     а) Указать имя (имя гостя), которое будет использоваться для поиска
        при визите;
     б) Указать действующий номер мобильного телефона для связи;
     в) Выбрать дату и время визита из доступных вариантов;
     г) Указать количество гостей (от 1 до 20);
     д) При необходимости — оставить комментарий с особыми пожеланиями
        (не более 500 символов);
     е) Подтвердить ознакомление с Пользовательским соглашением и Политикой
        конфиденциальности, поставив соответствующие отметки.

3.2. После отправки бронирования Пользователю отображается подтверждение
     с присвоенным номером бронирования. В случае если Пользователь дал
     согласие на получение уведомлений — дополнительно направляется
     сообщение в личные сообщения ВКонтакте.

3.3. Бронирование считается принятым с момента получения подтверждения
     от Приложения (статус HTTP 200 или отображение модального окна успеха).

3.4. Допускается только одно бронирование на один номер телефона в один
     и тот же день. При повторной попытке бронирования на ту же дату с
     тем же номером телефона Приложение выдаёт отказ с кодом 409.

________________________________________________________________________________

4.1. Пользователь вправе:

        настоящего Соглашения;
     б) Запрашивать у Кафе информацию о своём бронировании по телефону
        или лично;
     в) Отменить бронирование, связавшись с Кафе по телефону или через
        сообщество ВКонтакте;
     г) Обратиться за разъяснениями по вопросам обработки персональных
        данных в порядке, предусмотренном Политикой конфиденциальности.

4.2. Пользователь обязан:

     а) Предоставлять достоверные данные при оформлении бронирования;
     б) Использовать Приложение только в законных целях;
     в) Не осуществлять действия, направленные на нарушение работы
        Приложения (в том числе попытки автоматизированного перебора
        запросов, DDoS-атаки, внедрение вредоносного кода);
     г) Уведомить Кафе об отмене или изменении бронирования не позднее
        чем за 1 (один) час до времени бронирования;
     д) Соблюдать требования правил посещения кафе при визите.

4.3. Пользователю запрещается:

     а) Указывать заведомо ложные данные (имя, телефон);
     б) Оформлять бронирования с намерением не воспользоваться ими
        (намеренное «блокирование» столиков);
     в) Использовать технические средства для обхода системы ограничения
        запросов (rate limiting);
     г) Использовать Приложение лицам, не достигшим 14 (четырнадцати) лет,
        без согласия законных представителей.

________________________________________________________________________________

7.1. Приложение предназначено исключительно для личного некоммерческого
     использования физическими лицами.

7.2. Системой ограничения запросов установлены следующие лимиты:
     — не более 3 (трёх) запросов на бронирование с одного IP-адреса
       в течение 1 (одной) минуты;
     — не более 60 (шестидесяти) запросов от всех пользователей суммарно
       в течение 1 (одной) минуты.

     При превышении лимитов запрос отклоняется с кодом ответа 429.

7.3. Приложение не предназначено для оформления корпоративных мероприятий,
