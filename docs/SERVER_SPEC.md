# Технические характеристики приложения «Шоколадница»

> Актуально на: 18 марта 2026 г.
> Платформа: VK Mini App (бронирование столиков)

---

## 1. Архитектура системы

```
┌─────────────────────────────────────────────────┐
│              VK Mini App (браузер / WebView)     │
│         React 19 + VKUI 7 + VK Bridge           │
│              Сборка: Vite 7, ~757 KB             │
└──────────────────────┬──────────────────────────┘
                       │ HTTPS / API
┌──────────────────────▼──────────────────────────┐
│                   Сервер (VPS)                   │
│                                                  │
│  ┌─────────────┐  ┌──────────────┐  ┌─────────┐ │
│  │  FastAPI    │  │ VK Long Poll │  │Scheduler│ │
│  │  (uvicorn)  │  │  (vk_bot_   │  │(60s poll│ │
│  │  :8001      │  │  server.py)  │  │ loop)   │ │
│  └──────┬──────┘  └──────┬───────┘  └────┬────┘ │
│         │  asyncpg       │ psycopg2       │      │
│  ┌──────▼────────────────▼───────────────▼────┐  │
│  │           PostgreSQL (порт 5432)            │  │
│  └─────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────┘
                       │
              ┌────────▼────────┐
              │    VK API       │
              │  (сообщения     │
              │  гостям и       │
              │  официантам)    │
              └─────────────────┘
```

**3 самостоятельных процесса** запускаются через `start_all.py`:
| Процесс | Файл | Протокол |
|---|---|---|
| HTTP API | `app/main.py` | asyncpg (async) |
| VK Bot | `app/vk_bot_server.py` | psycopg2 (sync) |
| Scheduler | `app/scheduler.py` | psycopg2 (sync) |

## 6. Rate Limiting (защита от перегрузки)

| Тип | Лимит | Хранилище | TTL |
|---|---|---|---|
| Per-IP | 3 запроса / 60 сек | PostgreSQL | 120 сек |
| Глобальный | 60 запросов / 60 сек | In-memory (список) | 60 сек |
| Размер тела запроса | 64 KB | Middleware | — |

При превышении → HTTP **429 Too Many Requests**.

## 10. Что влияет на скорость — ключевые узкие места

| Узкое место | Влияние | Решение |
|---|---|---|
| VK API (отправка уведомлений) | +200–800 ms на бронь | Вынесено в фон — не блокирует ответ |
| PostgreSQL round-trip | 5–30 ms на запрос | Индексы на `phone+date`, `completed+scheduled_at` |
| asyncpg пул соединений | Очередь при > 30 запросов | Увеличить `pool_size` или добавить workers |
| Cold start Python | ~2–4 с при рестарте | systemd с `Restart=always`, preload |
| Scheduler poll | 60 с задержка задач | Допустимо для напоминаний и подтверждений |

## 11. Схема деплоя на VPS (рекомендуемая)

```
Internet
   │
   ▼
[Nginx]  ← SSL termination (Let's Encrypt, порт 443)
   │      ← Раздаёт статику фронтенда (dist/)
   │      ← Проксирует /api/* → localhost:8001
   ▼
[uvicorn :8001]   [vk_bot_server]   [scheduler]
        └──────────────┴──────────────┘
                       │
                [PostgreSQL :5432]
```

**Команды управления (systemd):**
```bash
systemctl start  shokoladnitsa
systemctl stop   shokoladnitsa
systemctl status shokoladnitsa
journalctl -u shokoladnitsa -f   # логи в реальном времени
```

## 12. Чеклист перед запуском на production

- [ ] Настроен `backend/.env` (DB, VK токен, адрес кафе)
- [ ] Применены миграции: `alembic upgrade head`
- [ ] Установлен и настроен Nginx с SSL (certbot)
- [ ] Все процессы запускаются через systemd с `Restart=always`
- [ ] Настроен firewall: закрыт порт 8001 снаружи, открыты 80/443
- [ ] Фронтенд собран: `npm run build` → файлы из `dist/` отдаёт Nginx
- [ ] Проверен `GET /api/health` — возвращает `{"status": "ok"}`
- [ ] (Опц.) Настроен `SENTRY_DSN` для мониторинга ошибок
