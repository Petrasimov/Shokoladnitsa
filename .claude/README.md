# Claude Code - Конфигурация проекта

Эта папка содержит настройки для удобной работы с Claude Code в проекте Shokoladnitsa.

## 📁 Файлы конфигурации

### `project-context.md`
Полное описание проекта:
- Архитектура и структура
- Стек технологий
- Паттерны разработки
- Важные соглашения

**Когда использовать**: Claude автоматически использует этот контекст для понимания проекта.

### `ignore`
Список файлов и папок, которые Claude должен игнорировать:
- node_modules, venv
- Build артефакты
- Временные файлы
- Конфиденциальные данные

**Когда использовать**: Автоматически применяется при поиске и чтении файлов.

### `hooks.sh`
Автоматические проверки и действия:
- Pre-commit хуки (линтинг)
- Post-read хуки (предупреждения)
- Pre-push хуки (проверка ветки)

**Когда использовать**: Настраивается для автоматического запуска.

### `commands.md`
Быстрый справочник команд:
- Запуск dev серверов
- Линтинг и форматирование
- Git workflow
- Работа с БД
- Отладка

**Когда использовать**: Справочник для частых операций разработки.

## 🚀 Быстрый старт

### 1. Запуск всего проекта

**Frontend:**
```bash
cd vk-table-booking
npm install
npm run dev
```

**Backend:**
```bash
cd backend
# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate

pip install -r requirements.txt
uvicorn app.main:app --reload
```

### 2. Работа с Claude Code

Claude будет автоматически:
- Понимать структуру проекта
- Игнорировать ненужные файлы
- Следовать паттернам проекта
- Предлагать правильные команды

### 3. Примеры запросов к Claude

**Создание новой фичи:**
> "Добавь новый эндпоинт для отмены бронирования в backend"

**Отладка:**
> "Помоги найти ошибку в telegram_bot.py"

**Рефакторинг:**
> "Оптимизируй queries в database.py"

**Git операции:**
> "Создай коммит с изменениями в frontend"

## 🎯 Рекомендации

1. **Всегда указывай контекст**: Frontend или Backend
2. **Используй конкретные имена файлов**: `telegram_bot.py`, а не "бот"
3. **Проверяй изменения перед коммитом**: Попроси Claude показать diff
4. **Используй линтеры**: Попроси запустить `npm run lint`

## 🔧 Настройка под себя

### Добавить новые игнорируемые файлы
Отредактируй `.claude/ignore`

### Добавить команды
Отредактируй `.claude/commands.md`

### Обновить описание проекта
Отредактируй `.claude/project-context.md`

### Настроить хуки
Отредактируй `.claude/hooks.sh` и сделай исполняемым:
```bash
chmod +x .claude/hooks.sh
```
